Runtime 类

Runtime 描述的是运行时的状态，也就是说在整个JVM之中，Runtime 是唯一一个与JVM运行状态有关的类并且默认提供一个该类的实例化对象

由于在每个JVM进程中只允许提供一个Runtime类对象，所以此类的构造方法被默认私有化，所以该类使用的是单例设计模式，并且单例设计模式一定有一个static方法获取本类实例

由于 Runtime类属于单例设计模式，如果要想获取实例化对象，那么就可以依靠类中的 getRuntime 完成

  • 获取实例化对象：public static Runtime getRuntime()

正则表达式

String 类型的数据在向其他数据类型转换的时候，为了保证转换的准确性，往往需要对其进行一些复杂的验证处理，那么这种情况下如果只是单纯的依靠 String 类中的方法，非常麻烦

例：判断字符串是否由数字组成，如果由数字组成则将其变为数字进行乘法计算

实际上这种功能是十分简单的，但如此简单的功能，却需要开发者编写大量的程序代码。

例：使用正则表达式实现同样的效果

正则表达式最早是从 Perl 语言里发展而来的，而后在JDK1.4以前如果需要使用到正则表达式的相关定义，则需要单独引入*.jar文件，但是从JDK1.4之后，正则已经默认被JDK所支持，并且提供由 java.util.regex开发包，同时对于 String类也进行了一些修改，使其可以有方法直接支持正则处理。

使用正则最大的特点在于方便进行验证处理，以及方便进行字符串的修改处理。

正则标记

从JDK1.4开始提供 java.util.regex 开发包，包中提供 Pattern程序类，在这个类中定义所有支持的正则标记

  1. 【数量单个】字符匹配
    • 任意字符：表示由任意字符组成；
    • \\：匹配\;
    • \n：匹配换行;
    • \t匹配制表符;
  1. 【数量：单个】字符集（可以从里面选取人一个字符）
    • [abc]：表示可能是字母a、b、c中的任意一个
    • [^abc]：表示不是由字母a、b、c中的任意一个组成
    • [a-zA-Z]：表示由任意字母组成，不区分大小写
    • [0-9]：表示由一位数字构成 ；
  2. 【数量：单个】简化字符集
    • .：表示任意一个字符;
    • \d：等价于[0-9]范围
    • \D：等价于[^0-9]范围
    • \s：匹配任意的一位空格，可能是空格、换行、制表符
    • \S：表示匹配任意的非空格数据
    • \w：匹配字母、数字、下划线，等价于[a-zA-Z 0-9]
    • \W：非匹配字母、数字、下划线，等价于[^a-zA-Z 0-9]
  3. 边界匹配：
    • ^：匹配边界开始
    • $：匹配边界结束
  4. 数量表达：默认情况下只有添加了数量单位才可以匹配多为字符；
    • 表达式?：改正则可以出现 0 次或 1次;
    • 表达式*：改正则可以出现 0 次、1 次或多次；
    • 表达式+：改正则可以出现 1 次或多次;
    • 表达式{n}：表达式的长度正好为 n 次
    • 表达式{n,}：表达式长度为 n 次以上;
    • 表达式{n,m}：表达式的长度在n~m次
  5. 逻辑表达式：可以连接多个正则：
    • 表达式 X 表达式 Y：X 表达式之后紧跟上外表达式
    • 表达式 X |表达式 Y：有一个表达式满足即可;
    • (表达式)：为表达式设置一个整体描述，可以为整体描述设置数量单位。

File类基本操作

在整个 java.io 包中，File 类是唯一一个于文件本身有关的类，而如果要想进行 File 类操作，（创建、删除、重命名等），必须提供完整的路径，而后才可以调用相应的方法处理。

File 类的基本使用：

File 类是 Comparable 接口的子类，所以 File 类对象是可以进行排序处理。在进行 File 类处理的时候需要设置访问路径，路径的配置主要通过File 类的构造方法处理；

  • 构造放方法：public File(String pathname)设置要操作的完整路径
  • 构造方法：public File(File parent,String child)设置父路径与子目录

如果想进行文件的基本操作可以使用如下的方法：

  • 创建新的文件：public boolean createNewFile() throws IOException;
  • 判断文件是否存在：public boolean exist();
  • 删除文件：public boolean delete();
