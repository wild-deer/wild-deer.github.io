二.java面向对象编程

1.面向对象简介

面向对象考虑的时模块化设计，而在使用的时候进行拼装。

面向对象的主要特征：

  1. 封装性:内部操作对外部而言不可见,当内部操作都不可直接使用的时候才算叫安全
  2. 继承性:在已有结构的基础上继续进行功能的扩充
  3. 多态性:在继承的基础上扩充而来的概念,指的是类型的转化处理

程序开发的三个步骤

OOA:面向对象分析

OOD:面向对象设计

OOP:面向对象编程

  • 栈内存：保存的是一块堆内存的地址。可以简单的理解为对象的名称保存在栈中

8.构造方法与匿名对象

可以通过构造方法实现实例化对象中的属性初始化处理

Java中构造方法的定义：

  • 构造方法名称必须与类名称保持一致
  • 构造方法不允许设置任何返回值类型。即没有返回值定义；
  • 构造方法是在使用关键字new实例化对象的时候自动调用的。

例：

①：主要是定义对象的所属类型，类型决定了可以调用的方法；

②：实例化对象名称，所用的操作通过对象来进行访问；

③：开辟一块新的堆内存空间；

④：调用有参构造。在Java程序里面考虑到程序结构的完整性，所以有的类都会提供构造方法，如果现在类中没有定义构造方法，那么会在程序编译的时候 自动创建一个什么都不做的无参的构造方法。但如果现在已经在类中定义构造方法时，这个默认的构造方法不会被默认创建；

所以一个类至少存在一个构造方法，永恒存在。

  • 既然构造方法时一个方法，为什么不定义返回类型？||既然构造方法不会返回数据，为什么不使用void定义？

    分析：程序编译器时根据代码结构来进行编译处理的，执行的时候也是根据代码结构来处理的。

    所以如果在构造方法上使用void，那么此结构就与普通方法结构完全相同了，这样编译器会认为次方法是一个普通方法。

  • 普通方法与构造方法最大的区别：构造方法实在类对象实例化的时候调用的，而普通方法是在类对象实例化产生之后调用的。

  • 构造方法重载时只需要考虑参数类型及个数即可。：在构造方法定义的时候最好根据参数个数进行安排定义顺序。

  • 匿名对象 ：
    定义对象的名称：类名称 对象名称 = null；

    实例化对象：对象名称 = new 类名称（）；

    此时也可以通过实例化对象来进行类的操作。由于这种形式的对象由于没有名字就称为匿名对象。

    由于匿名对象没有任何的引用名称，所以该对象使用一次之后就将称为垃圾，而所有的垃圾将被GC进行回收与释放。

9.this调用本类属性

this：当前对象

this可以描述以下三类结构:

  • 当前类中的属性：this.属性；
  • 当前类中的方法（普通方法、构造方法）this()、this.方法名（）；
  • 描述当前对象；

10.this调用方法

  • 构造方法调用(this()): 使用关键字new实例化对象的时候会调用构造方法
  • 普通方法的调用(this.方法名称()): 实例化对象产生后就可以调用普通方法

对于本类构造方法的相互调用需要注意:

  • 构造方法必须在实例化新对象的时候调用, 所以”this()”语句只允许放在构造方法首行
  • 构造方法互相调用时应保留程序出口

简单java类的核心开发结构

  • 类名称一定有意义，可以明确描述某一类事物
  • 类之中的所有属性都必须使用private进行封装，封装后的属性必须提供setter、getter
  • 类之中必须有无参构造方法
  • 类中不许出现任何输出语句，所有内容的获取必须返回
  • 【非必须】可以提供有一个获取对象详细信息的方法

15.代码块

在程序中使用“{}”定义的结构称为代码块

  • 普通代码块

特点：定义在一个方法之中的代码块。可以在一个方法中进行结构拆分，以防止相同变量名互相影响。

  • 构造代码块

特点：构造快会先于构造方法执行，且每一次实例化新对象时都会调用构造快中的方法

  • 静态代码块

静态代码块会优先于构造块执行，并只会执行一次，主要目的时为了金泰程序的初始化

静态代码块优先于主方法执行。
