8.构造方法与匿名对象

可以通过构造方法实现实例化对象中的属性初始化处理

Java中构造方法的定义：

  • 构造方法名称必须与类名称保持一致
  • 构造方法不允许设置任何返回值类型。即没有返回值定义；
  • 构造方法是在使用关键字new实例化对象的时候自动调用的。

例：

①：主要是定义对象的所属类型，类型决定了可以调用的方法；

②：实例化对象名称，所用的操作通过对象来进行访问；

③：开辟一块新的堆内存空间；

④：调用有参构造。在Java程序里面考虑到程序结构的完整性，所以有的类都会提供构造方法，如果现在类中没有定义构造方法，那么会在程序编译的时候 自动创建一个什么都不做的无参的构造方法。但如果现在已经在类中定义构造方法时，这个默认的构造方法不会被默认创建；

所以一个类至少存在一个构造方法，永恒存在。

  • 既然构造方法时一个方法，为什么不定义返回类型？||既然构造方法不会返回数据，为什么不使用void定义？

    分析：程序编译器时根据代码结构来进行编译处理的，执行的时候也是根据代码结构来处理的。

    所以如果在构造方法上使用void，那么此结构就与普通方法结构完全相同了，这样编译器会认为次方法是一个普通方法。

  • 普通方法与构造方法最大的区别：构造方法实在类对象实例化的时候调用的，而普通方法是在类对象实例化产生之后调用的。

  • 构造方法重载时只需要考虑参数类型及个数即可。：在构造方法定义的时候最好根据参数个数进行安排定义顺序。

  • 匿名对象 ：
    定义对象的名称：类名称 对象名称 = null；

    实例化对象：对象名称 = new 类名称（）；

    此时也可以通过实例化对象来进行类的操作。由于这种形式的对象由于没有名字就称为匿名对象。

    由于匿名对象没有任何的引用名称，所以该对象使用一次之后就将称为垃圾，而所有的垃圾将被GC进行回收与释放。

15.代码块

  • 普通代码块

特点：定义在一个方法之中的代码块。可以在一个方法中进行结构拆分，以防止相同变量名互相影响。

  • 构造代码块

41.子类实例化实现流程

子类实例化时会自动调用父类构造方法

实际子类构造方法隐藏了super()语句

super()表示子类调用父类构造语句,该语句只允许放在构造方法的首行

默认情况下的处理子类只会调用父类中的无参构造方法,所以写不写super()无影响

如果父类没有提供无参构造,就必须用super()明确调用有参构造

所以在实例化子类对象时会实例化父类对象,目的时为了所有的属性可以进行空间分配

super()和this()都可以调用构造函数

super()是由子类调用父类构造

this()是调用两类构造

两个方法都必须放在首行,所以不允许同时出现

48.多态

  • 方法多态性：

    • 对象向下转型模式（子类自己特殊的定义环境）：子类 子类实例 = （子类）覅类实例、强制完成转化

52.抽象类

抽象类的主要作用在于对子类中覆写方法进行约定，在抽象类中可以定义抽象方法实现这样的约定，抽象方法指的是使用了abstract关键字定义的并且没有提供方法体的方法，而抽象方法所在的类必须为抽象类，抽象类必须使用abstract关键字定义

例：定义一个抽象类

抽象类不是完整的类

抽象类的使用必须按照以下原则进行

  • 抽象类必须提供子类，子类使用extends继承一个抽象类；
  • 抽象类的子类（不是抽象类）一定要覆写抽象类中的全部抽象方法；
  • 抽象类的对象实例化可以利用对象多态性通过子类向上转型的方式完成。

例：

抽象类使用意见：

  • 抽象类无法直接实例化
  • 抽象类主要目的时进行过度操作使用，所以当使用抽象类进行开发的时候往往是设计中需要解决类继承问题时带来的代码重复

抽象类使用时需要注意的问题：

  • 定义时不能使用final关键字
  • 抽象类是普通类的基础上扩展而来的，抽象类会提供构造方法，并且子类也会按照子类对象的实例化原则进行父类构造调用。
  • 抽象类中允许没有抽象方法，也无法直接使用关键字new实例化抽象方法，必须依靠子类对象完成
  • 抽象类可以提供又static方法，并且该方法不受到抽象类对象的

static方法不受到实例化对象或结构的限制，可以直接通过类名称进行调用

54.包装类

基本数据类型并不是类，如果像将基本数据类型以类的形式处理，那需要对其进行包装

基本数据类型进行包装处理后可以像对象一样进行引用传递，同时也可以使用Object来接受。

所以在JDK1.0时提出了包装类的概念，提供八种包装类

JAVA中包装类分为两种类型：

  • 对象型包装类（Object直接子类）：Boolean、character；
  • 数值型的包装类（Number直接子类）：Byte、Short、Integer、Long、Float、Double

Number是一个抽象类，定义有如下方法

No 方法名称 类型 描述

3 public abstract int intValue() 普通 从包装类型中获取int数据
4 public abstract long longValue() 普通 从包装类型中获取long数据
5 public abstract float floatValue() 普通 从包装类型中获取float数据
6 public abstract double doubleValue() 普通 从包装类型中获取double数据

Number类中的方法就是直接提供有获取包装类中基本数据类型的功能，一共之定义有六个方法。

    • Integer类：public Intege(int value);

    • Boolean类：public Boolean(boolean value);

    • 数值型包装类已经由Number定义拆箱方法了；
    • Boolean型：public boolean booleanValue();
